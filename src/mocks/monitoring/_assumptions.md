# Monitoring Mock Fixtures — Assumptions & Interpolation Rules

## Fixture Scope

These fixtures serve SPRINT-07 Page 6 (Monitoring & Governance). They are read-only at load time
by adapters in `src/lib/api/monitoring/`. UI components MUST NOT import these JSON files directly.

## alerts.json

### Geometry
- All alert coordinates use kecamatan centroid geometry (point), NOT polygon bounds.
- The mini-map shows a single Leaflet marker per alert. Polygon display is post-PoC scope.

### In-memory mutation
- `getAlerts.ts` loads the fixture once, stores in an in-process variable `alertStore`.
- `patchAlertInStore()` mutates this variable shallowly — the imported JSON is never mutated.
- Alert status changes survive within the browser session but reset on hard refresh (intentional).

### Status ASSIGNED
- `ASSIGNED` is a PoC-only transient state added on top of the canonical OPEN/INVESTIGATED/RESOLVED/FALSE_POSITIVE set.
- Should be confirmed with product before real implementation. PoC uses it as the intermediate between OPEN and INVESTIGATED.

### FALSE_POSITIVE behavior
- Alert is hidden from the cards stack. Map marker is also hidden (simpler than strikethrough icon).
- The alert remains in the in-memory store with status FALSE_POSITIVE.

## monthly_snapshot.json

### Generation
- Generated by `scripts/generate_snapshots.ts` via linear interpolation (LERP).
- Run with: `npx tsx scripts/generate_snapshots.ts`
- Output is deterministic given the same anchor values.

### Date range
- Jan 2024 (index 0) through Apr 2026 (index 27) = 28 months × 3 profiles = 84 rows.
- Each row: `snapshot_date` = `YYYY-MM-01` (ISO-8601, always 1st of month).

### Interpolation strategy
- Linear interpolation: `value(i) = round(start + (end - start) * (i / 27))`
- All integer fields use `Math.round()`.
- `ndvi_score` uses 3 decimal places.
- `regulatory_flag` and `tren_wilayah_label` are fixed strings (no interpolation).

### harga_lahan_median vs wilayah_land_value_quarterly
- `harga_lahan_median` in monthly_snapshot uses a conservative valuation anchored
  to NJOP-based estimates from BPS Kab. Karo data.
- `land_value_quarterly.json` shows market listing price (Rp/m²) from DPMPTSP/OLX/Rumah123.
- Berastagi Q1 2024 listing price = 318,000 Rp/m² vs snapshot 180,000 — the difference
  reflects NJOP (assessed tax value) vs market listing price, consistent with
  the ~40-60% NJOP/market gap typical in rural highland areas.
- Apr 2026 both converge to ~420,000 as NJOP is updated to reflect the Tol Medan-Berastagi premium.

### Snapshot reconciliation drift tolerance
- Linear interpolation may produce location_score values that disagree with
  `computeLocationScore()` by ±2 points. This is documented and acceptable for PoC.
- If drift > 2 points is observed, override the snapshot value and document below.
- No overrides were required in the initial generation.

## policy_log.json

### Pre-seeded Berastagi entries (canonical per SPRINT.md §2.3)
1. `pol-ber-001`: Pembangunan Jalan Desa Cisondari — 2024-08-12 — infrastruktur
2. `pol-ber-002`: Pembangunan Tower 4G Naman Teran — 2024-11-05 — infrastruktur
3. `pol-ber-003`: MoU Koperasi Stroberi Semperjaya — 2025-02-18 — demand

### Attribution rows (pre-computed using heuristic)
- Cisondari road: Infrastructure +12 → 50% tinggi (sole infrastruktur policy in Q3 2024)
- Cisondari road: Market Access +12 → 30% sedang (multi-factor)
- Tower 4G: Infrastructure +6 → 30% sedang (multi policy period)
- MoU Koperasi: Demand Score +8 → 50% tinggi (sole demand policy in Q1 2025)

### localStorage schema
Key: `talis.policy_log.v1`
Value: `{ "version": 1, "items": [...PolicyLogItem[]] }`

### Policy tags taxonomy
- `infrastruktur` — roads, telecom, utilities, connectivity
- `demand` — cooperatives, market access, offtaker programs
- `regulasi` — zoning changes, RDTR, amdal decisions
- `alert_response` — auto-created on alert RESOLVED (TASK-006)

### Auto policy_log on alert RESOLVED (TASK-006)
- Created by `alertLifecycle.ts` via `addPolicyLog()`
- Title pattern: `"Resolusi Alert: {alert.tipe}"`
- tags: `["alert_response", alert.tipe_domain]`
- tipe_domain mapping:
  - konversi_lahan_ilegal → "regulasi"
  - kekeringan_parah / waspadai_kekeringan → "infrastruktur"
  - potensi_banjir → "infrastruktur"
  - penurunan_produktivitas → "demand"
- Auto-created entry is NOT removed when alert is re-opened (PoC simplification).

### addPolicyLog localStorage failure handling
- Catches `QuotaExceededError` and logs warning; does NOT throw.
- The lifecycle transition (alert RESOLVED) still completes even if localStorage write fails.
- Toast: "Alert resolved tapi entri kebijakan gagal disimpan — coba refresh."
